# ClawCode

OpenClaw-compatible agent system for Claude Code. Give Claude Code a persistent personality, memory, dreaming, and agent behavior — using the same format as [OpenClaw](https://github.com/openclaw/openclaw).

## What it does

This plugin turns Claude Code into a personal agent with:

- **Personality** — SOUL.md (core truths, vibe), IDENTITY.md (name, emoji), USER.md (your preferences)
- **Operational protocols** — AGENTS.md (session protocol, safety), TOOLS.md, HEARTBEAT.md
- **Memory search** — SQLite + FTS5 with BM25 ranking, temporal decay, MMR diversity
- **Optional QMD** — Local embeddings + vector search + reranking (no API keys)
- **Dreaming** — 3-phase nightly consolidation (Light/REM/Deep) that promotes memories to MEMORY.md
- **Heartbeats** — Periodic memory consolidation every 30 minutes
- **Bootstrap ritual** — Conversational onboarding where the agent discovers its identity
- **Import from OpenClaw** — Bring your existing agents, memory, and crons over
- **Hooks** — Pre-compaction flush, session summary, dream tracking
- **Compatible with other plugins** — Works alongside WhatsApp, Telegram, etc.

## How it works

The plugin runs an MCP server that reads your agent's bootstrap files (SOUL.md, IDENTITY.md, etc.) and injects them as `instructions` into Claude Code's context. Every conversation gets your agent's personality automatically.

Multiple plugins coexist — each MCP server's instructions are injected independently. So ClawCode (personality) + WhatsApp plugin (messaging) work together seamlessly.

## Prerequisites

- [Node.js](https://nodejs.org/) v18+
- Optional: [QMD](https://github.com/tobi/qmd) for enhanced semantic search

## Quick Setup

**1. Create a folder for your agent.**

Each agent lives in its own folder:

```sh
mkdir ~/my-agent && cd ~/my-agent
claude
```

**2. Install the plugin.**

```
/plugin marketplace add crisandrews/ClawCode
/plugin install agent@clawcode
```

**3. Create your agent or import from OpenClaw.**

Create from scratch (starts a conversational bootstrap ritual):
```
/agent:create
```

Or import an existing OpenClaw agent:
```
/agent:import
```

**4. Reload** to apply the personality:
```
/mcp reconnect clawcode
```

## Bootstrap Ritual

When you create a new agent, a `BOOTSTRAP.md` file acts as its "birth certificate". On first launch:

1. The agent sees BOOTSTRAP.md in its context
2. Starts a casual conversation: *"Hey. I just came online. Who am I? Who are you?"*
3. Together, you discover the agent's name, personality, vibe, and emoji
4. The agent writes IDENTITY.md, USER.md, and adjusts SOUL.md
5. Offers to set up QMD for enhanced memory (if available)
6. Deletes BOOTSTRAP.md — the agent is born
7. Runs `/mcp reconnect clawcode` to load its new identity

This is a one-time ritual. After that, the agent wakes up with its personality on every session.

## Memory System

### Two backends

**Builtin (default — no setup needed)**
- SQLite + FTS5 with BM25 ranking
- Temporal decay — dated files (`memory/YYYY-MM-DD.md`) lose relevance over time (30-day half-life)
- MMR — diversity re-ranking to avoid redundant results
- Markdown chunking — 400 tokens with 80 token overlap
- Keyword extraction with English + Spanish stop word filtering

**QMD (optional — enhanced search)**
- Local embeddings via node-llama-cpp — **no API keys needed**
- Vector search with semantic understanding (finds related concepts, not just exact keywords)
- Three search modes:
  - `search` — fast basic hybrid (vector + BM25)
  - `vsearch` — vector search with reranking (recommended)
  - `query` — full query expansion + rerank (slow, best quality)
- Requires [QMD](https://github.com/tobi/qmd) binary: `bun install -g qmd`
- Enable via `/agent:settings` or during bootstrap
- Falls back to builtin if QMD fails

### Memory lifecycle

**Daily logs** — Agent writes to `memory/YYYY-MM-DD.md` during sessions. Append-only.

**Pre-compaction flush** — When context nears the token limit, `PreCompact` hook reminds the agent to save important information before compression.

**Session summary** — `Stop` hook reminds the agent to write a conversation summary before closing. Includes: what was discussed, decisions made, open items.

**Heartbeat** — Every 30 minutes (auto-created cron), the agent reviews recent daily files and consolidates important items into `memory/MEMORY.md`. Respects active hours.

**Dream tracking** — Every `memory_search` call is recorded in `memory/.dreams/events.jsonl`. Each recalled memory gets scoring updates in `short-term-recall.json`: recall count, total score, concept tags, recall days.

### Dreaming

Nightly cron (3 AM, auto-created) runs a 3-phase memory consolidation:

**1. Light phase** — Ingests recent recall signals, deduplicates candidates, records reinforcement signals in `phase-signals.json`.

**2. REM phase** — Extracts recurring themes from concept tags, identifies multi-day patterns, writes `## REM Sleep` block to `DREAMS.md`. Records reinforcement signals for deep ranking.

**3. Deep phase** — Ranks all candidates using 6 weighted signals:

| Signal | Weight | What it measures |
|---|---|---|
| Relevance | 0.30 | Average retrieval quality |
| Frequency | 0.24 | How many times recalled |
| Query diversity | 0.15 | Distinct days the memory was searched |
| Recency | 0.15 | Time-decayed freshness (7-day half-life) |
| Consolidation | 0.10 | Multi-day recurrence strength |
| Conceptual richness | 0.06 | Concept-tag density |

Candidates must pass threshold gates (minScore, minRecallCount, minUniqueQueries). Snippets are rehydrated from live files (stale/deleted entries are skipped). Duplicates already in MEMORY.md are filtered. Winners are promoted (appended) to `memory/MEMORY.md` and a `## Deep Sleep` summary is written to `DREAMS.md`.

Run manually: `dream(action='run')` or `dream(action='dry-run')` to preview without writing.

## Configuration

Settings are stored in `agent-config.json`. View/modify via `/agent:settings`.

### Memory backend

```json
{
  "memory": {
    "backend": "builtin",
    "citations": "auto"
  }
}
```

For QMD:
```json
{
  "memory": {
    "backend": "qmd",
    "citations": "auto",
    "qmd": {
      "searchMode": "vsearch",
      "includeDefaultMemory": true,
      "limits": { "maxResults": 6, "timeoutMs": 15000 }
    }
  }
}
```

### Heartbeat active hours

Restrict heartbeats to your active window:

```json
{
  "heartbeat": {
    "schedule": "*/30 * * * *",
    "activeHours": {
      "start": "08:00",
      "end": "23:00",
      "timezone": "America/Santiago"
    }
  }
}
```

### Dreaming schedule

```json
{
  "dreaming": {
    "schedule": "0 3 * * *",
    "timezone": "America/Santiago"
  }
}
```

### Builtin search tuning

```json
{
  "memory": {
    "builtin": {
      "temporalDecay": true,
      "halfLifeDays": 30,
      "mmr": true,
      "mmrLambda": 0.7
    }
  }
}
```

After any config change: `/mcp reconnect clawcode` to apply.

## Hooks

Four lifecycle hooks are installed automatically:

| Hook | When | What it does |
|---|---|---|
| `SessionStart` | Session begins | Shows agent identity. Creates heartbeat + dreaming crons if missing. |
| `PreCompact` | Before context compression | Reminds agent to save important info to daily log. |
| `Stop` | Agent about to stop | Reminds agent to write session summary to daily log. |
| `SessionEnd` | Session closes | Logs session end event to dream tracking. |

## Importing from OpenClaw

`/agent:import` copies an existing OpenClaw agent into the current directory:

1. Lists available agents from `~/.openclaw/workspace*/`
2. Copies bootstrap files (SOUL.md, IDENTITY.md, USER.md, AGENTS.md, TOOLS.md, HEARTBEAT.md)
3. Optionally imports memory (MEMORY.md + recent daily files)
4. Adapts AGENTS.md for Claude Code (removes OpenClaw-specific tool references)
5. Reloads via `/mcp reconnect clawcode`

Credentials files are never copied.

## Importing crons from OpenClaw

`/agent:crons` converts OpenClaw crons (`~/.openclaw/cron/jobs.json`) to Claude Code local scheduled tasks:

- Cron expressions are kept as-is (local timezone)
- Prompts are adapted for Claude Code tools
- Delivery channels (WhatsApp) are mapped to MCP tool instructions
- `durable: true` — persist across Claude Code restarts (7-day expiration)
- Crons only run while Claude Code is open and REPL is idle

## Multiple agents

Each agent is its own folder. To switch agents: `cd ~/other-agent && claude`.

## Using with WhatsApp

Install [claude-whatsapp](https://github.com/crisandrews/claude-whatsapp) alongside ClawCode. Both plugins coexist — WhatsApp messages arrive with formatting rules AND the agent's personality active.

```
/plugin marketplace add crisandrews/claude-whatsapp
/plugin install whatsapp@claude-whatsapp
```

## Differences from OpenClaw

| Feature | OpenClaw | ClawCode |
|---|---|---|
| Runtime | 24/7 gateway daemon | Per-session (Claude Code) |
| Multi-channel | Native WhatsApp/Telegram/etc. | Via separate MCP plugins |
| Sub-agents | Persistent with own identity | Ephemeral (Claude Code Agent tool) |
| Heartbeats | Built-in every 30min | Local cron every 30min (auto-created) |
| Crons | Native with sub-second intervals | Local crons (durable, 7-day expiration) |
| Memory search | SQLite + FTS5 + embeddings | SQLite + FTS5 (+ QMD optional) |
| Dreaming | 3-phase (Light/REM/Deep) | 3-phase (Light/REM/Deep) |
| QMD | Built-in backend option | Optional via `/agent:settings` |
| Bootstrap | Conversational onboarding | Conversational onboarding |
| Voice/TTS | Built-in | Not included |
